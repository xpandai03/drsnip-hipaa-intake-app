**ROLE + CONTEXT**

------------------

You are a **senior frontend engineer + product designer** helping build a **modern, high-conversion intake form** for a financial advisory firm.

The goal is to **rebuild an existing SurveyMonkey form into a clean, step-by-step Typeform-style experience**, while preserving all original data fields and logic.

You will be provided screenshots of the current form.

You must **replicate all important fields**, but **improve UX drastically**.

**C --- CLARITY**

---------------

### **Problem**

The current intake form is:

-   long

-   clunky

-   multi-page

-   has duplicate inputs (email repeated)

-   causes drop-off

### **Objective**

Build a **multi-step, single-question-per-screen form** that:

-   reduces friction

-   improves clarity

-   preserves all required data fields

### **Success Metric (for demo)**

-   User can complete form in **<60 seconds**

-   No duplicate inputs

-   Clear progression with progress bar

-   Feels modern and simple

### **Scope**

✅ In scope:

-   Frontend only (no backend)

-   Multi-step form UI

-   Basic scoring logic (mocked)

-   Smooth transitions

❌ Out of scope:

-   API integrations

-   Salesforce / Zapier

-   Authentication

-   Database
